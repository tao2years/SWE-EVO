# 中文工程周报模板

## 精简版本

### 本周工作

一句话总结：概括本周最重要的产出，优先写“完成了什么能力或实验、解决了什么问题、验证到什么程度”。

- 工作项 1：一句话说明目标、关键动作和当前结果。
- 工作项 2：一句话说明目标、关键动作和当前结果。
- 工作项 3：一句话说明目标、关键动作和当前结果。
- 可选工作项 4：只有在确实重要时再写。

### 下周工作

- 计划 1：优先写近期可落地的验证项，例如多轮复测、bad case 复盘、参数收敛。
- 计划 2：优先写系统化建设项，例如评测系统、指标方案、看板展示、trace 接线。

默认不超过 2 条；只有用户明确要求更细时再展开。

## 当前项目推荐写法

- 完成 `project-coverage-7` 主链路接线与后台运行，打通推理、评测、汇总和看板展示。
- 对 `innercc` / `claude` 或不同 `innercc` 版本进行对比实验，基于 resolved、F2P、P2P、成本和 token 指标给出结论。
- 补齐看板统计与实验归档能力，使 run 结果可直接用于横向比较和后续复现。
- 修复评测或运行链路中的稳定性问题，确认不会阻塞批量实验。

## 多版本实验推荐写法

如果本周核心是版本对比，可以把一条工作项写成：

- 完成 `project-coverage-7` 上的多版本对比，覆盖 `版本名 / run id / resolved / 成本 / token / cache hit` 等关键指标，当前结论是……。

如果实验还没结束，改写成：

- 截至 YYYY-MM-DD，`版本名 / run id` 已完成 `X/Y` inference、`A/B` eval reports，当前观察到……，最终结论待补齐。
